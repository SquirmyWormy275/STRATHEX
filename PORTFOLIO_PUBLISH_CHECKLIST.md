# Portfolio publish checklist

Generated by `/portfolio-conversion`. These are actions only you can take. Work top to bottom.

## 1. Untrack files that are now gitignored

The skill added `CLAUDE.md`, `AGENTS.md`, `.cursor/`, `.aider*` to `.gitignore`. The `.claude/` directory was already gitignored. Files that were committed before they were gitignored stay tracked until you remove them from the index.

```powershell
git rm --cached CLAUDE.md
git rm --cached -r .claude/
git rm --cached .coverage          # if present
git status                          # verify the right files are staged for removal
```

The local files stay on disk (the `--cached` flag). Only the tracking is removed. Commit:

```powershell
git commit -m "Untrack CLAUDE.md, .claude/, and .coverage from public history"
```

If you also want to scrub these from past commits (so they're not reachable via `git log`), use `git filter-repo` or BFG Repo-Cleaner. **This rewrites history and force-pushes**, so only do it if you're sure no one is collaborating on the repo yet.

## 2. CLAUDE.md cross-references

## 3. Em-dashes in pre-existing wiki/docs

The skill removed em-dashes from the three new portfolio docs (README, ARCHITECTURE, CASE_STUDY) that hiring managers read first. The pre-existing wiki and docs have ~400 em-dashes. Those weren't authored by the skill and the skill scope was sanitization, not voice rewriting.

If you want consistent voice across the public surface, run a one-line PowerShell pass:

```powershell
Get-ChildItem -Recurse -Include *.md | ForEach-Object {
    (Get-Content $_.FullName -Raw) -replace ' — ', ', ' | Set-Content $_.FullName -Encoding UTF8
}
```

Spot-check a few files afterward. The replacement is mechanical and will sometimes produce slightly off prose (e.g. `, , ` if there was already a comma). Skim and fix.

## 9. Final pre-publish skim

Read [README.md](README.md) once more, top to bottom, as if you were a hiring manager seeing it for the first time. The first paragraph carries 80% of the skim weight. Make sure:

- The opening problem statement is concrete (it is: world-champion vs amateur, 25s vs 60s).
- The headline numbers are specific (0.3-0.8s spread, 2M Monte Carlo, < 2% win-rate spread).
- The "My role" paragraph is honest about scope (solo, plus the explicit citation that AAA rules and QAA tables are external).
- Every link works. Run a markdown link-checker if you have one (`markdown-link-check README.md`).


Open a PR against `main`, review your own diff, then merge. Make the repo public on GitHub if it isn't already.
